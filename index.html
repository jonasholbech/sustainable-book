Hvad er problemet?

I bund og grund er der kun to problemer

  1. Hver byte koster ressourcer at sende og modtage
  2. Kode kræver ressourcer at køre på serveren eller i browseren, jo mindre kode vi kører, dets bedre

Minimér ALT

Jo færre bytes du sender jo bedre! Hver byte koster på regnskabet.

Vi har rigtigt mange muligheder for at minimere den data vi sender over netværket. I HTML, CSS og JavaScript kan vi fjerne alle mellemrum der ikke har betydning, og det kan reducere størrelsen på vores filer betragteligt

Fjern alt hvad du kan fjerne, minimér din kode, dine billeder, dine videoer og minimer antallet af "requests"

Der findes rigtigt mange værktøjer der kan hjælpe med at minimere og med at fjerne "overhead". Astro (https://astro.build) er et nyt bud på hvordan vi kan udvikle på en moderne og effektiv måde, uden at sende alt muligt unødvendigt kode til brugeren

Vælg en grøn host

Vores projekter skal online, men dem der hoster vores sites er ikke alle lige grønne

Det kan være svært at vide hvad der er bedst da alle siger de er grønne, men undersøg sagen og tag et kig på projektet www.thegreenwebfoundation.org

Nettet ❤️ Statisk

Vores sites kan være dynamiske eller statiske.

Lidt forsimplet så er dynamiske site noget hvor indholdet hentes fra en database ved hvert eneste besøg

Statiske sites er sites hvor vi ikke behøver kode til at generere indeholdet. Det gør vi en gang for alle

Og så kan vi bare generere sitet igen hvis vores data ændrer sig

Hvis vi henter data ved hvert besøg, starter vi databasen op hele tiden. Kan vi undgå det kan vi spare super meget strøm, og vores sites bliver hurtigere, med bedre UX til følge

Sig nej til frameworks

Frameworks kan hjælpe med at bygge tingene hurtigere, men prisen bliver betalt af dine brugerre og af miljøet.

Kan du sige nej til frameworks, ekstra biblioteker og færdige designsystemer, kan du spare virkeligt mange bytes

Ikke alle frameworks er dog lige slemme, og frameworks så som Astro øger faktisk ikke størrelsen

Tænk dig om, kan du undvære det? Findes der et "lettere" alternativ?

Hvor mange af de tunge og vigtigste beslutninger bliver taget i design- og planlægningsfasen, så er det her et sted hvor kode folket kan tage nogle valg der gør en forskel

Eks: React fylder omkring 100kB plus din egen kode. Det er i sig selv nok ikke så problematisk. Problemet består i hvor let det er at smide endnu et framework oven i.

Tættere på brugeren

Når koden skal sendes til klienten, skal den sendes kortest muligt

Så, kan du vælge en host der er tæt på brugeren er det bedre i det store regnskab.

Når du bruger eks Supabase spørger den dig hvor din database skal være, rent fysisk, der er Frankfurt langt bedre for os end Washington

Til sidst findes der noget der hedder CDN du kan researche hvis du bruger frameworks

Mål, Mål, Mål

Mens du udvikler, og når dit site er live, så mål hvor mange kB dit site fylder

Det er så let at overse et billede så det pludseligt fylder 1-2MB

Google Lighthouse er et værktøj der kan give dig et bud på hvad du kan gøre bedre, men fanebaldet Netværk, i dine developer tools er meget let og det skal bare gøres til en vane at bruge det

Det er svært at sige præcis hvornår noget er for stort, men sæt dig et mål om at dit site aldrig må komme over 1MB, og gå efter det halve.

Brug det rigtige format

Når vi ikke kan komme uden om billeder og videoer, så brug det rigtige format

Når vi snakker billeder er formaterne AVIF og WebP som regel at foretrække

Når vi snakker video er WebM nok det bedste valg

https://squoosh.app/ er et super værktøj til at optimere dine billeder, men også her er der mulighed for noget automatisering

"Principle of least power"

Det er bedre at bruge HTML end CSS og CSS er bedre end JavaScript

Brug den mindst belastende teknologi til at løse dit problem

Kan det løses i HTML er det altid bedst, dernæst CSS og til sidst JavaScript

Kender du eksempelvis <details> elementet i HTML? Med 3 linjers html kan vi lave et element der kan åbnes og lukkes (tænk: Læs mere), UDEN brug af JavaScript

a11y

Accessibility (a11y) er et kæmpe område, som fortjener meget mere plads end det får her. Det knytter sig ikke til miljøet, men til slutbrugeren

Den kode du skriver påvirker dine brugere, og ikke alle har samme forudsætninger

Brug HTML som det var tiltænkt, links til at navigere, knapper til at interagere, lister, overskrifter og så videre, så får du en god start
